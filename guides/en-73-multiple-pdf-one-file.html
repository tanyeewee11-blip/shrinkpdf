The Right Order: Compress First, Then Merge

The most common mistake is merging all PDFs first and then trying to compress the large combined file. The correct order is the opposite:

✅ Correct workflow: Compress → Merge → Done
  1. Compress each PDF individually at the appropriate level
  2. Merge the compressed files into one combined PDF
  3. If the merged result is still over the limit, do one final compression pass
⚠️ Why merging first gives worse results

Different documents in your set may need different compression levels — your IC scan needs Maximum, your bank statement needs Balanced, your employment letter needs Light. If you merge first, you must apply one compression level to everything, which means over-compressing some documents (reducing quality unnecessarily) and under-compressing others. Compressing each file individually gives you precise control and the smallest possible total result.

Step-by-Step Workflow

1

List all documents you need to combine

Write down every file you need and what each one is (IC, payslips, bank statements, certs). This prevents accidentally missing a document after you've already merged everything.

2

Compress each file individually

Open ShrinkPDF, compress each PDF one at a time. Use the right level for each document type — see the table in the next section. Save each compressed file with a clear name (e.g., payslip_jan_compressed.pdf).

3

Merge all compressed files

Open the Merge PDF tool. Add all your compressed files in the order you want them to appear. Drag to reorder if needed. Click Merge.

4

Check the merged file size

The merged file size will be approximately the sum of all compressed files. If the portal limit is 5MB and your merged file is 4.2MB, you're done. If it's still over the limit, do one final compression pass on the merged file at Maximum.

5

Verify page order and content

Open the merged file and scroll through every page. Confirm: correct order, all documents included, all pages from each document present, no duplicates, all text legible.

Common Use Cases

Bank loan or credit card application

Banks typically ask for 3 months of payslips and 3 months of bank statements in a single PDF. Compress each month's document individually (Balanced for statements, Maximum for payslips if needed), then merge all 6 files. Most banks accept up to 5–10MB per submission.

LHDN tax filing

LHDN allows 2MB per document. If you need to submit multiple receipts for a single deduction category, merge them into one file first (after compressing each one), so you submit one combined file rather than many separate small files. This reduces the number of uploads and lowers the chance of missing a document.

Scholarship application

JPA, MARA, and Yayasan portals often accept one file per document category. Compress your result slip at Balanced, your IC at Maximum, your certs at Maximum. Then for the "Supporting Documents" category, merge certs and any recommendation letters into a single combined PDF.

FAQ

What if the portal only accepts one PDF with everything in it? ▼
Follow the compress-then-merge workflow exactly. Compress each document individually at the right level, merge in the correct order, and submit the combined file. If the single merged file exceeds the portal limit, do a final Maximum compression pass on the merged file. For very large document sets (10+ pages), the total may still exceed strict limits — in that case, contact the portal about submitting in batches or as separate files.
Does the page order matter when I merge? ▼
Yes, it often does. Portals and human reviewers expect documents in a logical order — typically: IC/identity document first, then income proof, then bank statements, then supporting documents. Check what order the portal or institution specifies. The Merge tool lets you drag files to reorder before merging.
Can I compress the merged PDF again after merging? ▼
Yes — if you've already compressed each file individually and the merged result is still slightly over the limit, a second compression pass on the merged file will reduce it further. The second pass compresses what was already compressed, so it won't reduce as dramatically as the first pass, but it's effective for shaving 20–40% off a nearly-compliant file.
