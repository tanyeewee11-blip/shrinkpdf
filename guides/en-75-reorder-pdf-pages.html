Does Reordering Affect Quality?

Reordering is lossless

Like merging and splitting, reordering works at the file-structure level rather than re-rendering anything. Each page is copied directly from the source document into its new position — same text, same embedded images at the same resolution, same formatting. Nothing is resampled, flattened, or recompressed. The only thing that changes between the original file and the result is the sequence the pages appear in.

This is a different operation from compression, which does rasterize pages to shrink file size. If you need both a specific page order and a smaller file, reorder first, then compress what's left — that way you're not compressing pages you're about to move or remove anyway.

Reordering and Deleting Pages Together

Documents that need reordering often also have pages that shouldn't be there at all — a blank page from a scanner misfeed, a duplicate, or a cover sheet that isn't needed in the final version. ShrinkPDF's reorder tool handles both in the same view: select any thumbnail and delete it, or drag pages into position, without switching tools or re-uploading the file partway through.

Made a mistake?

The tool keeps track of each page's original position, so you can reset the order back to how the file started if a drag goes wrong, without needing to re-upload.

Common Reasons to Reorder a PDF

Scanned documents out of sequence

Multi-page scans fed through a scanner or a phone scanning app sometimes come out in the wrong order, especially double-sided documents or stacks that got shuffled before scanning.

Moving a cover page, table of contents, or appendix

A report exported from Word or Canva sometimes places a title page, signature page, or appendix in the wrong spot — reordering fixes this without regenerating the document from the original source file.

Combining scans from multiple sessions

If a document was scanned in parts (front pages one day, back pages another) after merging the files, the pages may need reordering to read correctly from start to finish.

Reorder vs Split — Which Do You Need?

These solve different problems:

If the pages just need to be in a different order but should stay in one file, reorder. If the pages need to end up in separate files, split. If you have several source files that need to become one document in a specific order, merge them first, then reorder if the combined order still isn't right.

Frequently Asked Questions

Does reordering a PDF reduce the quality of the pages? ▼
No. Reordering copies each page directly from the source file into a new file structure — it doesn't re-render, rasterize, or recompress anything. Text stays selectable and images keep their original resolution; only the page order changes.
Can I delete pages while reordering? ▼
Yes. ShrinkPDF's reorder tool lets you select and delete pages in the same view as dragging pages into a new order, so you can clean up and rearrange a document in one pass instead of using two separate tools.
What's the difference between reordering and splitting a PDF? ▼
Reordering changes the sequence of pages within a single file. Splitting extracts pages out into separate files. If you need pages in a different order but still in one document, reorder. If you need pages separated into multiple files, split.
Is there a page limit for reordering a PDF? ▼
No. ShrinkPDF's reorder tool works on PDFs of any length, since the page thumbnails and drag-and-drop happen locally in your browser rather than on a server with file size restrictions.
Can I reorder PDF pages on my phone? ▼
Yes. The reorder tool supports touch drag on mobile browsers, so you can tap and drag page thumbnails to rearrange them the same way you would with a mouse on desktop.
Is it safe to reorder confidential documents? ▼
Yes. ShrinkPDF processes everything locally in your browser — files are never uploaded to any server during reordering, which matters for contracts, ID documents, or medical records.
